# Editing & Polish — Reference Guide

## Purpose

First drafts are never finished copy. This file covers the systematic editing process that transforms draft copy into polished, conversion-ready text. Each editing pass has a specific focus — running them in order catches different classes of problems.

## The 7-Pass Editing Process

### Pass 1: Readability

**Goal:** Make every sentence easy to scan and understand on first read.

| Check | Target | How to Fix |
|---|---|---|
| **Flesch-Kincaid Grade** | Grade 6-8 (consumer), 8-10 (B2B), 9-12 (technical) | Shorter sentences, simpler words, fewer clauses |
| **Sentence length** | No sentence over 25 words. Average 14-18 words. | Split long sentences at conjunctions. |
| **Paragraph length** | 1-3 sentences. Max 4 displayed lines. | Split at the second idea. One idea per paragraph. |
| **Passive voice** | Under 10% of sentences | "Our tool was designed by..." → "We designed..." |
| **Adverb density** | Remove unnecessary adverbs | "Extremely powerful" → "Powerful" (or better: specific proof) |
| **Syllable count** | Prefer 1-2 syllable words | "Utilize" → "Use". "Demonstrate" → "Show". "Approximately" → "About" |

**Tool:** Run copy through Hemingway Editor (hemingwayapp.com) or Readable.com. Target "Good" or better.

### Pass 2: Voice Consistency

**Goal:** Ensure all copy sounds like it was written by the same person/brand.

**Method:** Read the copy for 3 randomly selected pages in sequence. Flag any section where:
- Sentence length dramatically shifts (one section is all short punches, another is all long)
- Vocabulary register changes (one section says "leverage" and another says "use")
- Humor appears in one section but not others (humor must be consistent or absent)
- Person/perspective shifts (switching between "we" and "I" or "you" and "one")
- Formality level shifts (one section is casual, another is corporate)

**Fix:** Choose the voice that best represents the brief and revise outliers to match.

### Pass 3: Specificity

**Goal:** Replace every vague claim with a specific one.

**The Three Specificity Tests:**

| Test | How It Works | Example |
|---|---|---|
| **"Could a competitor say this?"** | If yes → too generic. Rewrite with your specific differentiator. | "We deliver results" → "847 clients. Avg. 2.4x increase in qualified leads." |
| **"So what?"** | Read each sentence and ask "so what?" If no compelling answer → cut or strengthen. | "We have years of experience" → so what? → "12 years. 200+ projects. 94% client retention rate." |
| **"Prove it"** | Every claim needs evidence. If you can't prove it, either find proof or cut the claim. | "Best-in-class service" → prove it: "4.9/5 on Clutch from 89 reviews" |

**Common vague → specific upgrades:**

| Vague | Specific |
|---|---|
| "Years of experience" | "12 years" or cut it |
| "Many clients" | "847 clients" |
| "Industry-leading" | "Ranked #1 on G2 for [category]" |
| "Fast turnaround" | "Delivered in 2 weeks, guaranteed" |
| "Affordable" | "Starting at $49/month" |
| "High-quality" | "Handmade from Italian leather" or specific quality marker |
| "A lot of features" | "47 integrations including Slack, Notion, and Salesforce" |
| "Excellent customer support" | "Average response time: 4 minutes. 97% satisfaction score." |

### Pass 4: Banned Word Elimination

**Goal:** Remove every word and phrase that signals lazy, generic, or AI-generated copy.

See `references/anti-patterns.md` for the full banned words list. Key categories:

**Corporate jargon to kill:**
leverage, synergy, ecosystem, empower, innovative, cutting-edge, state-of-the-art, seamless, robust, holistic, paradigm, streamline, scalable (unless genuinely about scale), disruptive, game-changing, best-in-class, world-class, next-generation

**AI-slop tells to kill:**
delve, tapestry, landscape ("in today's digital landscape"), crucible, multifaceted, myriad, plethora, embark, navigate (metaphorical), "it's important to note that", "at the end of the day", "in the realm of", "harness the power of"

**Weasel words to kill or strengthen:**
"up to" (commit to a real number), "helps" (say what it actually does), "may" (either it does or it doesn't), "virtually" (what does this add?), "solutions" (what solution? be specific)

### Pass 5: CTA Audit

**Goal:** Every page has a clear, strong primary CTA.

| Check | Fix |
|---|---|
| Does every page have a primary CTA? | Add one if missing |
| Is the CTA above the fold? | Move it up or add a hero CTA |
| Is the CTA specific and benefit-driven? | "Submit" → "Get My Free Report" |
| Is there CTA hierarchy? (one primary, secondary is subordinate) | Reduce visual weight of secondary CTAs |
| Does the CTA match the commitment level? | Don't ask "Buy Now" when they're still learning. Match the funnel stage. |
| Is risk reversal near the CTA? | Add "No credit card required" / "Cancel anytime" / "Money-back guarantee" |

### Pass 6: Proof Point Check

**Goal:** Every major claim is backed by evidence.

For each claim on each page, verify it has at least one of:
- A specific metric (number, percentage, dollar amount)
- A testimonial (with full attribution: name, role, company)
- A credential (certification, award, media mention)
- A case study reference (linked)
- A logical argument (if data isn't available)

**Mark claims without proof.** Either find proof, weaken the claim to what's provable, or cut the claim entirely.

### Pass 7: Rhythm Check

**Goal:** Copy flows when read aloud. No monotony, no stumbling.

**Method:** Read the copy out loud (or mentally vocalize it). Listen for:
- **Monotonous rhythm:** Same sentence length repeating. Fix by varying — short, medium, long, short.
- **Awkward phrasing:** Any sentence that makes you stumble. If you stumble, the reader bounces.
- **Missing transitions:** Sections that feel disconnected. Add a bucket brigade or transition sentence.
- **Run-on paragraphs:** Any paragraph that feels heavy. Split it.
- **Weak endings:** Pages that trail off. End with energy — a CTA, a bold statement, a callback to the opening.

## Revision Checklists

### Per-Page Checklist

```
[ ] H1 is clear and includes the primary value or keyword
[ ] Hero section communicates what, who, and why in under 50 words
[ ] Every feature is connected to a benefit
[ ] At least one proof point per major section
[ ] Primary CTA is visible and specific
[ ] No paragraph exceeds 4 displayed lines
[ ] No sentence exceeds 25 words
[ ] No banned words or phrases remain
[ ] Voice matches the brand brief
[ ] Page has a clear single primary purpose
```

### Full-Site Checklist

```
[ ] All pages reviewed through Passes 1-7
[ ] Navigation labels are consistent and clear
[ ] CTA text varies across pages (not the same button everywhere)
[ ] Voice is consistent across all pages
[ ] Social proof appears on at least 3 pages
[ ] SEO meta content is complete for every page
[ ] 404 page has helpful copy
[ ] Thank-you/confirmation pages have next-step copy
[ ] No orphan pages (every page links to the next logical step)
```

## Common Revision Scenarios

### "Make it shorter"

1. Cut the weakest sentences first (the ones that add no new information)
2. Combine two sentences that say similar things into one stronger sentence
3. Cut adverbs and unnecessary adjectives
4. Remove introductory phrases: "It's important to understand that..." → cut
5. **Never cut:** proof points, specific data, CTAs, objection handlers

### "Make it more [adjective]"

| Request | Concrete Copy Moves |
|---|---|
| "More professional" | Longer sentences, fewer contractions, more data, remove humor |
| "More casual" | Shorter sentences, contractions, conversational asides, address reader directly |
| "More confident" | Remove hedging words (might, could, possibly), use declarative sentences, lead with results |
| "More urgent" | Add deadlines, quantify the cost of inaction, use present tense, shorten sentences |
| "More human" | Add stories, use "you" and "we," include imperfections, show personality |
| "Bolder" | Shorter sentences, stronger verbs, imperative voice, provocative claims backed by proof |
