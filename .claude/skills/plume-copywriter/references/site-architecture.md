# Site Architecture — Reference Guide

## Purpose

Site architecture determines what pages exist, how they connect, and what job each page does. Get this wrong and copy ends up scattered across the wrong pages, with critical information buried and conversion paths unclear. Get it right and the copy practically writes itself — each page has a clear purpose and a single primary action.

## Page Structures by Site Type

### Portfolio / Personal Brand

| Page | Primary Job | Primary CTA | Key Copy Sections |
|---|---|---|---|
| Home | Establish who you are and what you do | View Work / Get in Touch | Hero (name + role + positioning), selected work grid, brief about teaser, testimonial |
| About | Build personal connection and credibility | Get in Touch | Origin story, process, values, skills/tools, fun facts |
| Work | Showcase capability through projects | View Project → Contact | Project grid with thumbnails + 1-line descriptions, filtering by type |
| Work/[project] | Prove quality on a specific project | Contact for Similar Work | Project overview, challenge, process, outcome, images, next project link |
| Contact | Convert interest into conversation | Send Message | Brief CTA paragraph, form, alternative methods, response time expectation |
| Blog (optional) | Demonstrate expertise and improve SEO | Read more → Contact | Post listing with excerpts, categories |

### Agency / Consultancy

| Page | Primary Job | Primary CTA | Key Copy Sections |
|---|---|---|---|
| Home | Establish positioning and credibility | View Services / See Our Work | Hero (value prop), services overview, selected work, social proof (logos, metrics), CTA |
| About | Humanize the agency, build trust | Get in Touch | Agency story, team bios, values/culture, approach/methodology |
| Services | Explain what you offer and why it matters | Get a Quote / Book a Call | Services overview, individual service details, process, pricing (if public) |
| Services/[service] | Deep-dive on a specific service | Request Proposal | Service description, benefits, process steps, related case studies, FAQ |
| Work | Prove you deliver results | View Case Study | Project grid, filtering by industry/service, brief outcome snippets |
| Work/[case-study] | Demonstrate specific results | Start a Project | Full case study (problem → process → outcome → metrics) |
| Blog | Thought leadership and SEO | Subscribe / Contact | Featured post, recent posts, categories |
| Contact | Convert to lead | Submit Inquiry | CTA copy, detailed form, office info, map (if relevant) |

### SaaS / Product

| Page | Primary Job | Primary CTA | Key Copy Sections |

| Pricing | Convert comparison shoppers | Choose Plan / Start Trial | Plan cards, feature comparison table, FAQ, guarantee, enterprise CTA |

### Startup / Landing Page

Single page with sequential sections:

| Section | Primary Job | Copy Length |
|---|---|---|
| Hero | Hook attention, state the value prop | Headline (5-12 words) + subheadline (15-30 words) + CTA |
| Problem | Agitate the pain point | 2-3 short paragraphs |
| Solution | Present the product as the answer | 1 paragraph + 3-4 key benefits |
| Features | Prove the solution works | 3-6 feature blocks with benefit-led headlines |
| Social Proof | Build trust | Testimonials, logos, metrics |
| Pricing (optional) | Remove price ambiguity | 1-3 plan cards |
| FAQ | Handle remaining objections | 5-8 questions |
| Final CTA | Close the deal | Headline + CTA (mirror hero CTA) |

## Information Architecture Principles

### The 3-Click Rule (Updated)

The old "3-click rule" is a myth — research shows users don't count clicks (Zoltán Gócza, UIE). What matters is **confidence at each step**. Each page must make it obvious:
1. Where am I?
2. What can I do here?
3. Where do I go next?

### Navigation Copy Rules

- **Main nav:** 5-7 items maximum. Use nouns (Services, Work, About) not verbs (Explore, Discover) — Nielsen Norman Group research shows nouns outperform verbs in navigation.
- **Labels:** One word when possible. Two words maximum. Avoid jargon the audience doesn't use.
- **Order:** Most important pages first (left), utility pages last (right). "Contact" is almost always last.
- **Dropdown menus:** Acceptable for service/product breakdowns. Label the parent clearly — don't use "More."

### User Flow Architecture

Map the conversion funnel before writing any copy:

```
Entry (Home / Landing / Blog Post)
  ↓
Interest (Services / Features / Case Study)
  ↓
Consideration (Pricing / Testimonials / FAQ)
  ↓
Decision (Contact / Signup / Purchase)
  ↓
Confirmation (Thank You / Welcome)
```

Every page must have a clear "next step." No dead ends. The primary CTA should always be visible without scrolling on desktop.

### Page Priority Matrix

Assign each page a priority tier before writing:

| Tier | Pages | Writing Order | Revision Depth |
|---|---|---|---|
| **Tier 1: Conversion-Critical** | Homepage, primary service/product page, pricing, landing pages | Write first | Full revision + A/B testing recommendations |
| **Tier 2: Trust-Building** | About, case studies, testimonials | Write second | Full revision |
| **Tier 3: Supporting** | Blog posts, FAQ, contact, utility pages | Write third | Light revision |
| **Tier 4: Utility** | 404, thank-you, coming soon, cookie policy | Write last | Voice consistency check only |
