# Anti-Patterns — Reference Guide

## Purpose

This file catalogs everything that makes web copy bad — corporate jargon, AI-generated copy tells, weasel words, structural mistakes, and the quality tests that catch them. Run these checks at every phase gate. If copy fails any quality test, revise before proceeding.

## AI-Generated Copy Tells

AI-written copy has recognizable fingerprints. Eliminating these is critical — readers and search engines increasingly penalize generic AI content.

### Vocabulary Tells

| AI-Slop Word/Phrase | Why It's a Tell | Replace With |
|---|---|---|
| "Delve" / "Delving into" | Massively overrepresented in AI output | "Explore", "look at", "dig into", or just cut it |
| "Tapestry" / "rich tapestry" | AI's favorite metaphor for complexity | Be specific about what's complex |
| "Landscape" ("in today's digital landscape") | Empty throat-clearing | Cut entirely or name the specific context |
| "Leverage" (as a verb) | AI default for "use" | "Use" |
| "Navigate" (metaphorical) | "Navigate the complexities of..." | Cut or be specific |
| "Multifaceted" | AI loves this word | "Complex" or better — name the specific facets |
| "Myriad" / "plethora" | Statistically overused by AI | "Many" or better — a specific number |
| "Embark on a journey" | AI narrative cliche | Cut. Just start. |
| "Harness the power of" | AI hype language | "Use" |
| "Crucible" | Very rare in natural writing, common in AI | Don't use unless writing about metallurgy |
| "Revolutionize" / "transform" | AI default for any improvement | Be specific about what changes and how much |
| "Seamless" / "seamlessly" | AI's favorite adverb | Cut or describe the actual experience |
| "Elevate" | AI default for "improve" | "Improve" or specific: "increase by X%" |
| "It's important to note that" | AI filler preamble | Cut — just say the thing |
| "In conclusion" / "To sum up" | AI signals it's wrapping up | Cut — end with a strong statement or CTA |
| "Foster" (as in "foster innovation") | AI-favorite for "encourage" | "Encourage" or "build" |
| "Spearhead" | AI-favorite for "lead" | "Lead" |
| "Bolster" | AI-favorite for "strengthen" | "Strengthen" or "support" |
| "Realm" / "in the realm of" | AI filler | Cut — name the specific domain |
| "Testament" / "a testament to" | AI narrative cliche | Show the evidence directly |
| "Pave the way" | AI metaphor for "enable" | "Enable" or "make possible" |
| "At its core" | AI throat-clearing | Cut — just say the thing |
| "When it comes to" | AI transition filler | Cut or restructure the sentence |
| "Unlocking the power of" | AI headline pattern | Use a specific benefit instead |

### Structural Tells

| Pattern | Why It's a Tell | Fix |
|---|---|---|
| Every paragraph is exactly 3 sentences | Unnatural uniformity | Vary paragraph length: 1, 2, 3, 4 sentences |
| Every section follows the same structure | Template feel | Vary section structures — some with bullets, some narrative, some with tables |
| Opening with a definition ("X is...") | AI default opening move | Open with a hook, question, or bold claim instead |
| Lists always have exactly 5 items | AI defaults to 5 | Use the natural number — sometimes 3, sometimes 7, sometimes 4 |
| "It's not just about X, it's about Y" | AI's favorite contrast structure | Rewrite. Sometimes it IS just about X. |
| Emoji or exclamation marks at the start of every bullet | AI pattern in casual registers | Remove or use sparingly |
| Every paragraph ends with a generalization | AI wraps up each thought with a platitude | Cut the final sentence of each paragraph — if nothing is lost, it was a platitude |

### Tone Tells

| Pattern | Fix |
|---|---|
| Relentlessly positive — no tension, no trade-offs, no honesty about limitations | Add honest moments: "This isn't for everyone. If [condition], [alternative] might be better." |
| Every sentence is roughly the same emotional intensity | Vary intensity. Let some sentences be quiet. Build to peaks. |
| Using hedge phrases that sound diplomatic but say nothing | Cut the hedge. Commit to a position. |

### 2025-Era AI Copy Patterns

The AI content landscape has evolved beyond early ChatGPT tells. Watch for these newer patterns:

| Pattern | Why It's a Tell | Fix |
|---|---|---|
| **Every blog post opens with a definition** ("X is a...") | AI's default opening move for any topic | Open with a hook, story, or bold claim |
| **"Comprehensive guide" / "ultimate guide" in every title** | AI overuses these modifiers | Use specific, descriptive titles |
| **Perfectly balanced pros/cons** (always exactly 3 of each) | Real analysis is messy — pros and cons aren't symmetric | Let the actual analysis determine the count |
| **Generic "real-world examples" that are actually hypothetical** | "Imagine a small business owner named Sarah..." | Use actual examples with real names, or don't use examples |
| **Ending every section with a forward-looking statement** | "As we move forward..." or "As this trend continues..." | End with the conclusion, not a prediction |
| **Suspiciously perfect structure** — every H2 has exactly 3 H3s, every section the same length | Real writing has uneven structure | Let content depth determine structure |
| **AI hedging disguised as nuance** | "While there's no one-size-fits-all answer..." (before giving a generic answer) | Either commit to advice or explain the specific variables |
| **Excessive transitional smoothness** | No rough edges, no personality, no moments of friction | Add opinion, acknowledge trade-offs, be direct |

## Corporate Jargon

### Banned Words List

These words should trigger a rewrite. They communicate nothing specific:

**The Jargon Graveyard:**

| Word | Why It's Dead | Alternative |
|---|---|---|
| Synergy | Says nothing specific | Name the specific collaboration or benefit |
| Solutions | Everything is a "solution" — meaningless | Name what you actually provide |
| Innovative | Self-applied "innovative" is never convincing | Show the innovation with a specific example |
| Cutting-edge | Same as "innovative" — empty self-praise | Describe what's actually new |
| Best-in-class | Unsubstantiated superlative | Cite the ranking, award, or data |
| World-class | Same problem | Same fix |
| Ecosystem | Overused to the point of meaninglessness | Name the specific components |
| Empower | Vague and patronizing | Name the specific capability you're giving them |
| Holistic | Pretending to mean "comprehensive" | "Complete" or list what's actually included |
| Paradigm shift | Hyperbolic and dated | Describe the actual change |
| Disruptive | Self-applied disruption is cringe | Let others call you disruptive. Describe what you changed. |
| Game-changing | Same problem | Same fix — show the change with data |
| Streamline | Vague process improvement claim | "Cuts [process] from 3 hours to 20 minutes" |
| Optimize | Often means "we did something" | Name what improved and by how much |
| Robust | Meaningless filler adjective | Describe the specific capability |
| Scalable | Often used generically | "Handles 10 to 10M records without configuration changes" |
| Next-generation | Says "new" without saying how | Describe what's actually different |
| State-of-the-art | Empty superlative | Name the specific technology or approach |

## Weasel Words

Words that let copy avoid making real, verifiable claims:

| Weasel Word | Problem | Fix |
|---|---|---|
| "Up to" | "Up to 50% faster" could mean 1% faster | Commit: "35% faster on average" or show the range |
| "As much as" | Same hedging | Use the actual number |
| "Helps" | "Helps you grow" — how much? how? | "Grows your list by an average of 2.4x" |
| "May" / "might" | "May improve results" — or may not | Either it does or it doesn't. State the result. |
| "Virtually" | "Virtually eliminates errors" = still has errors | "Reduces errors by 97%" |
| "Solutions" (used vaguely) | "We provide marketing solutions" — what solutions? | "We write conversion copy and run A/B tests" |
| "Various" / "numerous" | Lazy counting | Use the actual number |
| "Significant" | "Significant improvement" — 5%? 500%? | State the percentage |

## Common Web Copy Mistakes

### Mistake 1: Writing About Yourself Instead of the Customer

| Self-Centered | Customer-Centered |
|---|---|
| "We are a leading provider of..." | "Get [outcome] without [pain]" |
| "Our team has 20 years of experience" | "Your project benefits from 20 years of [specific] expertise" |
| "We pride ourselves on quality" | "Every deliverable is reviewed 3 times before it reaches you" |

**The "We/You" ratio test:** Count "we/our" vs. "you/your" on the homepage. The ratio should be at least 2:1 in favor of "you/your." Many sites are inverted.

### Mistake 2: Feature-Dumping Without Benefits

Every feature needs a "so you can" bridge:
- Feature only: "Real-time analytics dashboard"
- Feature + benefit: "Real-time analytics dashboard — so you catch problems before they cost you customers"

### Mistake 3: Burying the Value Proposition

The value proposition must be in the hero section — the first thing a visitor sees. If the best copy is in the third section, move it up.

### Mistake 4: Weak CTAs

| Weak | Strong | Why |
|---|---|---|
| "Submit" | "Get My Free Report" | Describes the reward |
| "Click Here" | "See Pricing" | Describes where they'll go |
| "Learn More" | "See How It Works" | More specific about what they'll learn |
| "Contact" | "Book a Free 15-Min Call" | Reduces ambiguity and friction |

### Mistake 5: Wall of Text

No visual breaks, no subheadings, no bullet points. Fix:
- Maximum 3-4 lines per paragraph
- Subheading every 200-300 words
- Bullet points for any list of 3+ items
- Pull quotes for testimonials
- Bold key phrases for scanners

## Quality Tests

### The 5-Second Test

Show someone the homepage for 5 seconds. Then ask:
1. What does this company do?
2. Who is it for?
3. What should I do next?

If they can't answer all three, the copy needs work.

### The Competitor Swap Test

Take any sentence from the copy and put a competitor's name on it. If it still works, the sentence is too generic to be effective.

### The Deletion Test

Read through the copy and delete every sentence that seems weak. If the copy still makes sense (or improves), those sentences were noise.

### The Grandma Test

Could your non-technical grandmother understand what the company does from the homepage? If not, the jargon level is too high for most audiences.

### The "Why Should I Care?" Test

For every section, ask "why should the reader care about this?" If the answer isn't immediately obvious, either add the "why" or cut the section.
