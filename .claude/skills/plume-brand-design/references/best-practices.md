# Best Practice Sources — Master Reference List

## Purpose

This file catalogs the most authoritative, freely available resources for each domain of brand and web design. Use these to verify decisions, deepen knowledge, and cite rationale.

## Tier 1: Essential References (Read These First)

### Design Systems Documentation
| Source | URL | Why It Matters |
|--------|-----|---------------|
| **Material Design 3** | m3.material.io | The most comprehensive public design system. Gold standard for color, typography, motion, layout, and component documentation. |
| **Apple Human Interface Guidelines** | developer.apple.com/design/human-interface-guidelines | Best-in-class guidance on platform-native patterns, color, typography, and motion. |
| **IBM Carbon Design System** | carbondesignsystem.com | Open-source enterprise design system with excellent documentation. |
| **GOV.UK Design System** | design-system.service.gov.uk | The benchmark for accessible, well-documented component systems. |
| **Radix UI** | radix-ui.com | Headless components with exceptional accessibility and color system. |

### Brand Design Process
| Source | URL / Reference | Why It Matters |

| **Marty Neumeier, "The Brand Gap"** | Book | Foundational strategic thinking on brand as perception, not assets. |
| **Collins (agency)** | wearecollins.com | Case studies showing strategy → creative platform → identity system pipeline. |

### Books (Essential Reading List)
1. "Refactoring UI" — Wathan & Schoger (practical, developer-focused)
2. "The Design of Everyday Things" — Don Norman (cognitive science)
3. "Thinking with Type" — Ellen Lupton (typography fundamentals)
4. "Grid Systems in Graphic Design" — Josef Müller-Brockmann (layout mastery)
5. "Branding: In Five and a Half Steps" — Michael Johnson (brand process)
6. "The Brand Gap" — Marty Neumeier (brand strategy)
7. "Atomic Design" — Brad Frost (component methodology, free online)
8. "Don't Make Me Think" — Steve Krug (usability, quick read)
